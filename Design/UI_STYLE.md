# UI STYLE — Дизайн-система интерфейса

> Диегетический **CRT-терминал тюремного надзора**. Интерфейс выглядит как
> внутриигровые мониторы блока: зелёный люминофор на тёмном бетоне, изношенные
> gunmetal-рамки, трафаретные подписи, уголковые скобки, мягкие скан-линии.
> Это дизайн-двойник [ART_STYLE.md](ART_STYLE.md): UI и мир говорят на одном языке.
>
> **Код — единственная реализация этого документа.** Токены живут в
> [`Assets/Scripts/UI/UITheme.cs`](../Assets/Scripts/UI/UITheme.cs), сборка
> компонентов — в [`Assets/Scripts/UI/UIKit.cs`](../Assets/Scripts/UI/UIKit.cs).
> Экраны НЕ хранят собственных цветов/кеглей/отступов — только берут их отсюда.

---

## 1. Принципы

1. **Диегетичность.** Любая панель — это «экран» или «табличка» тюрьмы, а не
   абстрактный прямоугольник. Рамка, скобки, трафарет, скан-линии — обязательный
   «хром».
2. **Один акцент.** Единственное семейство акцента — зелёный люминофор
   (`#82eb96` / `#58c470`). Больше НЕТ золотого, циана и трёх разных «accent» в
   разных файлах (историческая причина «ИИ-слопа»).
3. **Ничего ярче 1.0.** Состояния кнопок — буквальные цвета, без хака
   `Color(1.45, 1.45, 1.45)` (пере-экспонированный белый).
4. **Читаемость важнее украшений.** Скан-линии и свечение — тонкие; текст всегда
   контрастный люминофор на тёмном.
5. **Холодная палитра.** Тени зеленят; тёплое только у редких «опасных»
   ржаво-красных индикаторов.
6. **Токен, а не число.** Любой отступ/кегль/цвет — из шкалы, не «магическое»
   значение по месту.

## 3. Типографика

Шрифт — **терминальный/пиксельный** (моноширинный, кириллица обязательна),
грузится один раз через `UIKit.Font` из `Resources/Fonts/`. Пока файл не
импортирован — безопасный откат на встроенный шрифт. Никакого сглаживания «в
кашу»: точечная фильтрация, чтобы пиксель оставался хрустящим.

| Роль | Кегль (px @1280×720) | Применение |
|---|---|---|
| `TypeDisplay` | 40 | заголовок экрана |
| `TypeTitle` | 28 | заголовок панели, имя говорящего |
| `TypeHeader` | 22 | заголовок секции |
| `TypeBody` | 18 | основной читаемый текст |
| `TypeLabel` | 15 | трафаретные подписи, хинты клавиш |
| `TypeCaption` | 12 | микротекст HUD, сноски |

Трафаретные подписи — ВЕРХНИМ регистром (`UIKit.CreateStencilLabel`).

## 4. Ритм и сетка

Шкала отступов на базе 4px: **4 / 8 / 12 / 16 / 24 / 32** (`Space1…Space8`).
Всё выравнивание, паддинги и промежутки — только из неё.

Рамки: `BorderThin 1` · `BorderMed 2` · `BorderThick 3`. Уголковые скобки:
плечо `18`, толщина `2`.

Слои (замена `GUI.depth` на `Canvas.sortingOrder`): HUD `100` · WorldMarkers
`200` · Experiment `800` · Map `900` · Board `950` · Dialogue `1000` ·
Journal `1100`.

## 5. Компоненты (анатомия)

Все создаются `UIKit.*`; ниже — что это и из чего собрано.

- **Terminal panel** (`CreateTerminalPanel`) — базовая единица. Заливка `Panel`
  + двухслойная рамка (`Chrome` снаружи, `Border` внутри) + уголковые скобки
  (`ChromeHi`) + опц. скан-линии. Возвращает уже вписанный `Content`-прямоугольник.
- **Button** (`CreateButton`) — состояния из токенов, `fadeDuration = MotionFast`.
  В покое `PanelRaised`, наведение — светлее, нажатие — яркий `Accent`.
  Отключённая — `ButtonDisabled`. Текст-подпись не перехватывает клики.
- **List row** (`CreateListRow`) — строка списка; фон (`RowNormal`/`RowActive`/
  `Selected`) ведёт вызывающий, чтобы отразить выбор/активность.
- **Tab bar** (`CreateTabBar`) — лента вкладок равной ширины.
- **Meter** (`CreateMeter` + `SetMeter`) — шкала (тревога/прогресс/HP): тёмный
  колодец + заливка, уровень задаётся через якорь (без спрайта).
- **Key-hint chip** (`CreateKeyHintChip`) — «[E] ДЕЙСТВИЕ»: клавиша акцентом,
  действие — трафаретом.
- **Stencil label** (`CreateStencilLabel`) — приглушённая подпись верхним регистром.
- **Toast** (`CreateToast`) — маленькое всплывающее уведомление в терминальной рамке.
- **World marker** (`CreateWorldMarker` → `WorldMarker`) — глиф «?/!», шкала
  тревоги и/или подпись, следящие за мировым объектом. Централизует прежнюю
  OnGUI-математику `WorldToScreenPoint`; в отличие от OnGUI — **виден в
  headless-скриншоте**.

## 6. «Хром» CRT

- **Скан-линии** — один тайлящийся спрайт `Resources/Sprites/ui_scanline.png`
  (точечная фильтрация, wrap repeat, `raycastTarget=false`), одна отрисовка на
  канвас. Опц. прокрутка UV со скоростью `ScanlineSpeed` через
  `Time.unscaledDeltaTime` (чтобы не замирало при `timeScale=0`).
- **Уголковые скобки** — тонкая геометрия (`Image`), без текстур.
- **Свечение** — не пост-обработка (не трогаем URP-рендер мира). Достигается
  ярким цветом люминофора и опц. мягким спрайтом `ui_glow.png` за акцентами.
- **Рамка** — двухслойная: внешний `Chrome` (изношенный металл) + внутренняя
  тонкая `Border` (зелёная линия).

---

## 7. Движение

Всё — в НЕмасштабированных секундах (модальные экраны ставят `timeScale=0`):
нажатие `MotionFast 0.08` · затухание панели `MotionMed 0.16` · открытие экрана
`MotionSlow 0.30`.

---

## 8. Рендер и headless

Канвасы строятся как **`ScreenSpaceCamera`** (`UIKit.CreateRootCanvas`,
`worldFacing:true`), а не Overlay: только такой канвас снимается headless-камерой
(`RenderPipeline.SubmitRenderRequest`, см. `SmokeCaptureTests`). Это же делает UI
проверяемым визуальными регрессионными тестами. `ScaleWithScreenSize`,
опорное `1280×720`, match `0.5`.

---

## 9. Do / Don't

**Do:** брать цвет/кегль/отступ из токена · оборачивать контент в
`CreateTerminalPanel` · держать один зелёный акцент · трафарет верхним регистром ·
считать анимации через `unscaledDeltaTime`.

**Don't:** хардкодить `new Color(...)` в экране · множить акценты (золото/циан) ·
пере-экспонировать состояния (`>1.0`) · класть сырой жёлтый/красный (`(1,0.85,0.1)`,
`(1,0.2,0.15)`) — только `Warning`/`Danger` · строить панель без рамки/скобок ·
использовать Overlay-канвас (не попадёт в скриншот).
