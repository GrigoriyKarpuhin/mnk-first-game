# Инструкции для LLM-агентов

## Локальный рабочий roadmap

Используйте `Design/MY_GROUP_CURRENT_ROADMAP.md` как локальный рабочий контекст
для текущей дизайн- или разработческой проблемы.

Этот файл намеренно игнорируется git. Поддерживайте его полезным для текущей
работы:

- фиксируйте текущую решаемую проблему;
- фиксируйте активные предположения и нерешённые вопросы;
- фиксируйте следующее конкретное дизайн- или техническое решение;
- обновляйте файл, когда меняется фокус или появляется принятое решение.

Не воспринимайте этот файл как источник подтверждённого game design. Это
временная рабочая память для активной совместной работы.

## Дизайн-документы

Файлы в `Design/`, которые отслеживаются git, должны содержать принятые решения,
стабильные спецификации, контракты и контекст, который команда готова разделять.

Не записывайте учебные задания, сырые наброски брейншторма или приватные
рассуждения в отслеживаемые дизайн-документы. Когда из текущей работы появляется
решение, обновляйте соответствующий отслеживаемый дизайн-документ уже принятым
результатом.

## Правила пользователя

- Не откатывайте изменения, которые сделали не вы.
- Синхронизируйте дизайн-документы с принятыми решениями.
